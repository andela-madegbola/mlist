    Introduction

    mlist-api is an API service that allows users create bucketlists to store items. Bucketlists are initialized with a name. Items are stored under bucketlists, with a name and a done-status indicating whether item has been completed or not. For full access to the API, a user account is required. After registration, an initial request is made to log in to the user account. This request generates a JSON Web Token, which is returned in the response. This token is used to authenticate subsequent requests to the API.

    Endpoints data is returned in JSON format, and whenever you make a request to an endpoint that does not exist, we do our best to handle such scenairos by returning a freindly JSON formated error message.

    You can checkout our GitHub repository if you want to contribute or raise an issue.

    Available End Points

    NOTE: The API is currently at version 1, so ensure you prefix each endpoint with “api/v1/” to access them.
    End Point Function
    POST /users: Create a new user
    POST /auth/login: Logs in a user
    GET /auth/logged_in: Shows the status of the user
    GET /auth/logout: Logs out a user
    POST /bucketlists: Creates a new bucket list
    GET /bucketlists: Lists all the created bucket lists
    GET /bucketlists/(id): Gets a single bucketlist
    PUT /bucketlists/(id): Updates this single bucketlist
    DELETE /bucketlists/(id): Deletes this single bucketlist
    POST /bucketlists/(id)/items: Creates a new bucketlist item
    GET /bucketlists(id)/items: Fetch all bucketlist items
    GET /bucketlists/(id)/items/(item_id): Fetch a single bucketlist item
    PUT /bucketlists/(id)/items/(item_id): Updates a bucketlist item
    DELETE /bucketlists/(id)/items/(item_id): Deletes an item in a bucketlist

    Authentication

    Creating a User

    HTTP Status: 201 

    curl -i -X POST http://mlist-api.herokuapp.com/api/v1/users \
-d "username=mayowa&email=mayowa@gmail.com&password=password"

    The above should return JSON data in the format 

    
  {
    "status": "User created successfully, Login to receive a token"
  }

    To use our API, you will have to create an account simply by sending a POST request to the appropriate endpoint

    HTTP Request

    POST http://mlist-api.herokuapp.com/api/v1/users

    Query Parameters

    Parameter Description
    username a valid username
    email a valid email
    password a valid string with a minium of 6 characters
    Remember request must be of type POST

    User Login

    HTTP Status: 200 

    curl -i -X POST http://mlist-api.herokuapp.com/api/v1/auth/login \
-d "username=mayowa&password=password"

    The above should return JSON data in the format 

    
  {
    "token": "eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9.eyJkYXRhIjp7InVzZXJfaWQiOjV9LCJleHAiOjE0Nzc2NDk5NDh9.OYA0Rbx_08hlahcsby5qr8r1HES5BOy3V6y115pnmBI"
  }

    HTTP Request

    POST http://mlist-api.herokuapp.com/api/v1/auth/login

    After successfull account creation, you will be automatically logged-in and assigned a token. This token would be used for subsequent request sent to the server.

    Query Parameters

    Parameter Description
    username a valid username
    password a valid string with a minimum of 6 characters
    Note that your token expires automatically after every 3 hours of logging in, you may need to login again after 3 hours to continue using the API.

    User Logout

    HTTP Status: 200

    curl -i -X GET http://mlist-api.herokuapp.com/api/v1/auth/logout \
-H "token: eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9.eyJkYXRhIjp7InVzZXJfaWQiOjV9LCJleHAiOjE0Nzc2NDk5NDh9.OYA0Rbx_08hlahcsby5qr8r1HES5BOy3V6y115pnmBI"

    The above should return JSON data in the format 

      {
    "message": "Successfully logged out"
  }

    HTTP Request

    GET http://mlist-api.herokuapp.com/api/v1/auth/logout

    When you are done, you may want to make your token expire instantly, simply send this GET request to the server

    Headers

    Header Description
    token a valid user token
    Please ensure you pass in a valid user token or you may not be able to make it expire.
